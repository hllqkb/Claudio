Task: Add Play History page + Favorites system

## Context
- Fastify backend at apps/server/src/, React frontend at apps/web/src/
- SQLite DB: `plays` table has (id, item_id, item_type, song_id, action, scene, created_at)
- SQLite DB: `songs` table has (id, source, title, artist, album, cover_url, duration_ms, updated_at)
- plays.song_id JOIN songs.id gives song details
- plays.repo.ts already has recordPlay(), upsertSong(), getTopArtists() etc.

## Task A: Play History API

In `apps/server/src/routes/player.ts`, add these endpoints:

```
GET /api/plays/recent?limit=50
```
Query:
```sql
SELECT p.id, p.song_id, s.title, s.artist, s.cover_url, p.scene, p.action, p.created_at
FROM plays p LEFT JOIN songs s ON p.song_id = s.id
ORDER BY p.created_at DESC LIMIT ?
```
Return: `{ plays: [...] }` with camelCase keys (songId, coverUrl, createdAt)

```
GET /api/favorites
```
```
POST /api/favorites
DELETE /api/favorites/:songId
```

For favorites, first check if favorites table exists in apps/server/src/db/schema.sql. If not, add it:
```sql
CREATE TABLE IF NOT EXISTS favorites (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  song_id TEXT NOT NULL UNIQUE,
  title TEXT,
  artist TEXT,
  cover_url TEXT,
  created_at TEXT DEFAULT CURRENT_TIMESTAMP
);
```

Add to plays.repo.ts:
- getRecentPlays(limit): returns play history with song info
- getFavorites(): returns all favorites
- addFavorite(songId, title, artist, coverUrl): insert
- removeFavorite(songId): delete

Register new routes in playerRoutes or create a separate file.
